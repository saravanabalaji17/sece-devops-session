Here's a hands-on Kubernetes lab to understand **HostPath → PV → PVC → Pod** flow.

---

# Lab Objective

Create:

1. HostPath directory on the node
2. Persistent Volume (PV)
3. Persistent Volume Claim (PVC)
4. Pod using the PVC
5. Validate data persistence

---

# Step 1: Create HostPath Directory

Login to the worker node where the pod will run.

```bash
mkdir -p /data/hostpath-pv
echo "Hello from HostPath PV" > /data/hostpath-pv/test.txt

ls -l /data/hostpath-pv
cat /data/hostpath-pv/test.txt
```

Expected:

```text
test.txt
```

---

# Step 2: Create Persistent Volume

Create `pv.yaml`

```yaml
apiVersion: v1
kind: PersistentVolume
metadata:
  name: hostpath-pv
spec:
  capacity:
    storage: 1Gi
  accessModes:
    - ReadWriteOnce
  persistentVolumeReclaimPolicy: Retain
  hostPath:
    path: /data/hostpath-pv
```

Apply:

```bash
kubectl apply -f pv.yaml
```

Validate:

```bash
kubectl get pv
```

Expected:

```text
NAME          CAPACITY   ACCESS MODES
hostpath-pv   1Gi        RWO
```

---

# Step 3: Create PVC

Create `pvc.yaml`

```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: hostpath-pvc
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 500Mi
```

Apply:

```bash
kubectl apply -f pvc.yaml
```

Validate:

```bash
kubectl get pvc
kubectl get pv
```

Expected:

```text
PVC STATUS = Bound
PV STATUS  = Bound
```

Describe:

```bash
kubectl describe pvc hostpath-pvc
```

Verify bound PV:

```text
Volume: hostpath-pv
```

---

# Step 4: Create Pod Using PVC

Create `pod.yaml`

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: hostpath-pod
spec:
  containers:
  - name: nginx
    image: nginx
    volumeMounts:
    - name: data
      mountPath: /usr/share/nginx/html
  volumes:
  - name: data
    persistentVolumeClaim:
      claimName: hostpath-pvc
```

Apply:

```bash
kubectl apply -f pod.yaml
```

Validate:

```bash
kubectl get pod
```

Expected:

```text
hostpath-pod   Running
```

---

# Step 5: Verify Existing Data

Check inside pod:

```bash
kubectl exec -it hostpath-pod -- ls /usr/share/nginx/html
```

Expected:

```text
test.txt
```

Read file:

```bash
kubectl exec -it hostpath-pod -- cat /usr/share/nginx/html/test.txt
```

Expected:

```text
Hello from HostPath PV
```

---

# Step 6: Write Data from Pod

```bash
kubectl exec -it hostpath-pod -- sh
```

Inside pod:

```bash
echo "Data written from Pod" > /usr/share/nginx/html/podfile.txt
exit
```

Verify:

```bash
kubectl exec -it hostpath-pod -- cat /usr/share/nginx/html/podfile.txt
```

---

# Step 7: Verify on Node

Login to node.

```bash
ls -l /data/hostpath-pv
cat /data/hostpath-pv/podfile.txt
```

Expected:

```text
Data written from Pod
```

This proves:

```text
Node Directory
      ↓
HostPath PV
      ↓
PVC
      ↓
Pod
```

---

# Step 8: Test Persistence

Delete pod:

```bash
kubectl delete pod hostpath-pod
```

Recreate:

```bash
kubectl apply -f pod.yaml
```

Check file:

```bash
kubectl exec -it hostpath-pod -- cat /usr/share/nginx/html/podfile.txt
```

Expected:

```text
Data written from Pod
```

Data persists because it is stored in the PV, not inside the container.

# Validation Commands Summary

```bash
kubectl get pv
kubectl get pvc
kubectl describe pv hostpath-pv
kubectl describe pvc hostpath-pvc
kubectl get pod
kubectl exec -it hostpath-pod -- ls /usr/share/nginx/html
kubectl exec -it hostpath-pod -- cat /usr/share/nginx/html/test.txt
kubectl exec -it hostpath-pod -- cat /usr/share/nginx/html/podfile.txt
```

### Additional Exercise

Try creating:

* PV size = 2Gi
* PVC request = 1Gi
* Second PVC requesting 3Gi

Observe why the second PVC remains in **Pending** state. This helps understand PV-PVC binding rules.
